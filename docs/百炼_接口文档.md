# 阿里云百炼（DashScope）接口整理 — 与本项目相关

本文根据阿里云帮助中心公开文档整理，便于后端用 **HTTP**（非 SDK）接入。权威说明以官方为准。

- [获取 API Key](https://help.aliyun.com/zh/model-studio/get-api-key)
- [地域说明](https://help.aliyun.com/zh/model-studio/regions/)

---

## 一、本项目采用的方案：Qwen-ASR（OpenAI 兼容模式 + Base64）

前端上传的音频保存在本机 `Storage`，**无公网 URL**。百炼 **Fun-ASR / Paraformer 录音文件识别** 要求 `file_urls` 为公网可访问地址，因此本项目选用 **千问 Qwen-ASR** 的 **OpenAI 兼容接口**，支持 **Data URL（`data:<mime>;base64,...`）** 传入音频（官方文档「输入内容：Base64 编码的音频文件」）。

### 1. 端点（中国内地）

| 项目 | 值 |
|------|-----|
| 方法 | `POST` |
| URL | `https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions` |
| 国际地域 | `https://dashscope-intl.aliyuncs.com/compatible-mode/v1/chat/completions`（需使用对应地域的 API Key） |

### 2. 请求头

| 头 | 说明 |
|----|------|
| `Authorization` | `Bearer <API_KEY>` |
| `Content-Type` | `application/json` |

### 3. 请求体要点

- `model`：例如 `qwen3-asr-flash`（以控制台/文档当前模型名为准）。
- `stream`：`false`（非流式，便于一次取全文）。
- `messages`：OpenAI Chat 结构；`user.content` 为数组，内含一项：
  - `type`: `input_audio`
  - `input_audio.data`: 公网 **URL** 或 **Data URL**（`data:audio/wav;base64,...` 等）。
- `asr_options`：与官方示例一致，例如 `enable_itn`（是否逆文本规范化）。

### 4. 输入大小与格式（官方约束摘要）

- Base64 后需满足平台对 **输入音频大小** 的限制（文档示例为 **10MB** 量级），请在产品侧控制录音时长/码率。
- 支持多种音频容器/编码；浏览器常见 **WebM（含 Opus）** 可使用 Data URL，`MIME` 可使用 `video/webm`（与容器一致）。

### 5. 响应（非流式）

- 与 OpenAI `chat/completions` 类似：从 `choices[0].message.content` 读取识别文本（字符串）；若为多段结构，需按文档解析。

### 6. 参考文档

- [Qwen-ASR API 调用方式及输入输出参数](https://help.aliyun.com/zh/model-studio/qwen-asr-api-reference)

## 二、备选方案：Fun-ASR 录音文件识别（仅公网 URL）

适用于音频已托管在 **OSS / CDN** 等可公网访问地址的场景。

### 1. 提交任务

| 项目 | 值 |
|------|-----|
| 方法 | `POST` |
| URL（中国内地） | `https://dashscope.aliyuncs.com/api/v1/services/audio/asr/transcription` |
| 国际 | `https://dashscope-intl.aliyuncs.com/api/v1/services/audio/asr/transcription` |
| 请求头 | `Authorization: Bearer <API_KEY>`，`Content-Type: application/json`，**`X-DashScope-Async: enable`**（异步提交，不可省略） |
| 消息体 | `model`、`input.file_urls`（URL 数组）等 |

### 2. 约束摘要

- **不支持**本地文件直传或 Base64；**仅** `file_urls`（`http/https`，或 REST 下 OSS 的 `oss://` 临时 URL 等，见官方说明）。
- 单次最多 100 个 URL；单文件大小、时长见官方上限。

### 3. 参考文档

- [Fun-ASR 录音文件识别 RESTful API](https://help.aliyun.com/zh/model-studio/fun-asr-recorded-speech-recognition-restful-api)

## 三、开发规范与安全（摘要）

1. **API Key** 仅放服务端，由 `backend/.env` 提供；勿写入前端仓库。
2. **勿在前端直连** DashScope；由本后端中转（官方亦要求通过服务端调用）。
3. 调用前若存在全局 HTTP 代理环境变量，可能影响链路；本项目在发起百炼 HTTP 请求前 **清除代理相关环境变量**，并对 HTTP 客户端使用 **`trust_env=False`**，避免库自动走系统代理。

---

## 四、本项目环境变量（`backend/.env`）

| 变量 | 说明 |
|------|------|
| `DASHSCOPE_API_KEY` | 百炼 API Key |
| `DASHSCOPE_CHAT_COMPLETIONS_URL` | 默认北京兼容端点，国际部署可改 |
| `ASR_MODEL` | 默认 `qwen3-asr-flash` |
| `ASR_ENABLE_ITN` | `true` / `false` |
| `ASR_TIMEOUT_SECONDS` | HTTP 超时（秒） |
| `ASR_MAX_PAYLOAD_BYTES` | Data URL 最大字节数（与官方 Base64 上限对齐，默认 10MB） |

识别完成后，原始 HTTP JSON 与提取的文本会写入 `Storage` 下 `*.asr.json`（与音频同主文件名）。

## 五、CosyVoice 语音合成（非流式 HTTP）

流水线在 **DeepSeek 整合话术** 之后，使用 **CosyVoice** 将答复转为语音。与 ASR **共用** `DASHSCOPE_API_KEY`。

### 1. 端点（中国内地）

| 项目 | 值 |
|------|-----|
| 方法 | `POST` |
| URL | `https://dashscope.aliyuncs.com/api/v1/services/audio/tts/SpeechSynthesizer` |
| 请求头 | `Authorization: Bearer <API_KEY>`，`Content-Type: application/json`；非流式**不要**带 `X-DashScope-SSE` |

### 2. 请求体（摘要）

- `model`：如 `cosyvoice-v3-flash`、`cosyvoice-v3-plus` 等（以控制台为准）。
- `input.text`：待合成文本。
- `input.voice`：音色（参见官方音色列表）。
- `input.format`：如 `mp3`、`wav`。
- `input.sample_rate`：如 `22050`。

### 3. 响应（非流式）

- `output.audio.url`：完整音频下载地址（**有时效**，本项目会 **立即下载** 并保存到 `Storage` 下 `*.tts.mp3`（或所选格式））。
- `output.audio.data`：非流式下通常为空，以 `url` 为准。

### 4. 参考文档

- [CosyVoice 语音合成 API 参考（非实时）](https://help.aliyun.com/zh/model-studio/non-realtime-cosyvoice-api)

## 六、环境变量补充（TTS / 公网回放）

| 变量 | 说明 |
|------|------|
| `COSYVOICE_TTS_URL` | 默认见上 |
| `TTS_MODEL` / `TTS_VOICE` / `TTS_FORMAT` / `TTS_SAMPLE_RATE` | 合成参数 |
| `TTS_TIMEOUT_SECONDS` / `TTS_MAX_CHARACTERS` | 超时与最长文本 |
| `BACKEND_PUBLIC_ORIGIN` | 返回给前端的 `audio_url` 前缀（需与浏览器访问后端一致，如 `http://127.0.0.1:8000`） |
